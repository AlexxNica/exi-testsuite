Profile Interoperability Test Report

This report was generated from the EXI profile interoperability test suite. The 3 tested implementations are EXIficient, Canon and EXIP. The table presents results for combinations of decoder/encoder (e.g., the column labeled "Canon2EXIficient" shows results for all test cases encoded by implementation "Canon" and decoded by implementation "EXIficient").

At the time of writing EXIP library only supports encoding and decoding of EXI Profile streams with the most constrained set of Profile parameter values. That is maximumNumberOfBuiltInElementGrammars set to 0, maximumNumberOfBuiltInProductions set to 0 and localValuePartitions parameter set to 0. Including these Profile parameters in the EXI header as described by the EXI Profile specification is not supported i.e. only out-of-band negotiation of these values is implemented. Additionally, the following restrictions also apply - preserving processing instructions, comments and lexical values are not supported and compression, pre-compression and self-contained elements are also not supported. That is the reason why one can see N/A entries in the table.

Note: The Working Group also maintains a list of publicly available implementations on its home page.
